Inbred Line Variant Database Help 1.0

Here is a brief manual on how to use the database:

Step 1: Specify interested variants

The first step is to specify the interested variants. ILVDB provides three ways to do so: by region, by variant_id, or by specific genes.

Region: Select Region in the Select-input-type selectfield, specify chromesome and start/end postion of the region, all variants in the region will be collected for further analysis. The reference genome is mm9.

Varient_id: Select Varient_id in the Select-input-type selectfield, type interested ilvdb-varient_ids to specify the variants. The GUI also takes a series of ids seperated by comma.

Gene: Select Gene in the Select-input-type selectfield, type interested Gene Ensembl ID to specify the gene, all the variants being annotated will be collected . The GUI also takes a series of Ensembl Ids seperated by comma. The Gene ID Converter link is helpful to convert your gene index to Ensembl Id.

Step 2: Specify interested view

The second step is to specify interested view. In the Select-view-type selectfield, the user can query about genotype, diplotype and the genotype and diplotype of the simulated F1 crosses. Each of these view types includes different output collums, which can be further arranged in the Select-included-fields multiselectfiled.

For genotype and diplotype tables,the Inbred-Strains multiselectfiled is used to specify interested strains. All 60 CC strains and 8 founders is available.

For genotype cross and diplotype cross tables, the user need to specify the two founder strains to simulate F1 crosses.

Step 3: Add further conditions

Paste the query and search the database.
