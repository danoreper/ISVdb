Description

ILVDB (Inbred Line Variant Database) is a database to store Collaborative Cross variant alleles and allele characteristics. It stores 1ï¼‰microarray genotyping that provides a probabilistic estimate of the founder haplotype mosaic for each CC line; 2) Inbred line sequencing that provides the variants within each CC founder haplotype; 3) Founder haplotype information per variant position, per CC line; 4) The ancestral subspecies per variant, per CC line etc. In order to help users more effiently get information from ILVDB, this Query and GUI interface is made. In the GUI interface the user only need to define the region they want to query, set constrictions and then they can get the data table or download the data.

Requirements and Installation

Installing Requirement

The installing of the database requers the follows:

  1. Get connection permission for vpn.unc.edu, Group: RC-Cust
  2. Get ssh permission for onyen@valdarlab.its.unc.edu
  3. Get login permission for MySql database at onyen@valdarlab.its.unc.edu
  4. Install R and Rstudio; install R packages MySQL and Shiny.

    install.packages('RMySQL', type='source')

    install.packages('shiny')

  5. In Rstudio set the RUN APP options to Run External.
  6. Have a ssh command line tool ready. Examples: Termial for Mac and Putty for windows
  7. Get the ILVDB_Shiny package code from Valdarlab.

Set up process

  1. Connect to RC-Cust at vpn.unc.edu. (Rcommended Software: CISCO AnyConnect)
  2. At your commend line tool, run:

    ssh -nNT -L 3308:localhost:3306 onyen@valdardb.its.unc.edu

    Keep the window opened and connected. This commend will forward romote data to your local interface by tunnel 3308-3306

  3. Open the code server.r and ui.r from ILVDB_Shiny package in Rstudio, set Rstudio work directory to the directory of the package.
  4. Press run external. You should see a user interface pump out.

Command line options

The huge table download function works best on local commend-line style. Here is the step to do this:

  1. Use ILVDB-Shiny GUI to generate the MySQL code, cope the code
  2. At your commend line tool, run:

    ssh onyen@valdardb.its.unc.edu "mysql --user='root' --password='${password}' --database='exon_1410b' -e '${MySQL_code}';" > ${local_output_file}

    This command will connect to the database and run MySQL query at the remoter server and return to local when it is done. Replace the value of ${} parameters with your target. The command requires a large menmory for big tables.
